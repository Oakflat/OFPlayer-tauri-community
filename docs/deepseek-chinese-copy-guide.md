# DeepSeek 中文文案调整指南

这份文档给 DeepSeek 用来集中调整 OFPlayer 的中文用户文案。目标是让界面更像产品，而不是像开发日志。

## 修改范围

优先修改：

- `src/composables/useI18n.js` 中中文区块的用户可见文案。
- 组件里少量硬编码的中文文案。
- 用户文档里的操作说明和提示文案。

谨慎修改：

- Rust 返回给前端显示的错误文案。可以改得更易懂，但不要影响错误判断逻辑。
- 英文文案。除非中文调整需要同步概念，否则先不动。

不要修改：

- i18n key 名称、对象层级和代码逻辑。
- `{count}`、`{shown}`、`{total}` 这类占位符。
- 命令、环境变量、文件路径、协议名示例本身。
- 开发文档里的必要技术说明，除非它会直接出现在 UI 中。

## 总原则

用户界面只说用户能理解和能行动的内容。技术细节放到诊断、日志和开发文档里。

优先写：

- 状态：正在连接、正在整理、已完成、需要重试。
- 行动：检查地址、重新连接、稍后再试、选择目录。
- 结果：已加入曲库、封面会继续整理、没有找到歌曲。

避免写：

- `PROPFIND`、`Subsonic API`、`endpoint`、`payload`、`base64`、`SQLite`、`Tauri`、`Rust`。
- “索引”“缓存”“元数据”等工程词，除非在设置页的高级说明里。
- 过长的因果解释。
- 责备用户的语气，比如“你填写错误”。

## 语气

- 简短、清楚、温和。
- 不卖弄技术，不解释实现。
- 不把内部模块名暴露给用户。
- 能一句说清就不要两句。
- 按钮文案用动词开头，比如“连接”“重试”“清理”“关闭”。

## 推荐词表

| 技术表达 | 用户表达 |
| --- | --- |
| endpoint | 服务器地址 |
| root path | 音乐目录 |
| folder id | 目录编号 |
| metadata | 歌曲信息 |
| artwork | 封面 |
| index / indexed | 加入曲库 / 已加入曲库 |
| cache | 本地副本 / 临时文件 |
| diagnostics | 问题报告 / 体验诊断 |
| sync | 同步 / 整理 |
| failed | 未完成 / 失败 |
| WebDAV endpoint rejected PROPFIND | 这个地址不是可用的 WebDAV 共享地址 |
| Subsonic API | Navidrome 连接 |

保留这些专有名词：

- OFPlayer
- WebDAV
- Navidrome
- FTP

## 示例

不好：

> 连接真正支持 PROPFIND 的 WebDAV 端点，例如 /dav。

好：

> 连接自托管的共享音乐目录。

不好：

> 已识别到 Navidrome 服务。OFPlayer 会使用 Subsonic API，并忽略 /app 这类网页路径。

好：

> 已识别为 Navidrome，已帮你修正地址。

不好：

> 已索引 {count} 首远端曲目，元数据和封面会按需补齐。

好：

> 已加入 {count} 首远程歌曲，歌曲信息和封面会在后台继续整理。

不好：

> 社区版不包含诊断上传端点。

好：

> 问题报告暂时不可用，请在开发设置中补充上传地址。

## 分场景规则

### 连接外部曲库

用户关心的是能不能连上、应该填什么、下一步做什么。

- “服务器地址”比“端点”好。
- “音乐目录”比“根路径”好。
- 自动修正只说结果，不说算法。
- 报错优先给修复方向：检查地址、账号密码、网络、服务类型。

### 扫描和同步

不要把扫描阶段讲成数据库或索引任务。

- “正在整理曲库”
- “正在加入远程歌曲”
- “歌曲信息和封面会在后台继续整理”

### 存储和清理

设置页可以稍微具体，但仍要避免数据库字段和内部路径。

- “曲库数据”优于“曲库数据库”
- “封面文件”优于“去重封面图片资产”
- “本地副本”优于“缓存”

### 诊断和隐私

把“诊断”解释成“帮助定位问题”，避免像遥测。

- 说明不会收集曲名、文件路径、歌词。
- 上传失败不要暴露环境变量，普通用户只需要知道“暂时不可用”。
- 开发环境可以在括号或高级说明里保留配置线索。

## 检查清单

提交前检查：

- `npm run build` 通过。
- 没有改动 i18n key 和对象结构。
- 所有 `{placeholder}` 都保留。
- 搜索中文区块里是否还出现不必要的技术词：

```powershell
rg -n "PROPFIND|Subsonic API|endpoint|payload|base64|SQLite|Tauri|Rust|元数据|索引|缓存" src\composables\useI18n.js
```

允许保留的情况：

- 用户明确选择的协议或服务名，例如 WebDAV、Navidrome、FTP。
- 设置页高级信息。
- 开发文档、诊断日志、代码注释。
