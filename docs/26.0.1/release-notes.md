# OFPlayer v26.0.1 Release Notes

日期：2026-05-17
类型：首个小版本更新 / 打包签名候选

## 版本定位

`26.0.1` 是 `26.0.0` 首发后的第一个小版本更新，重点处理真实用户测试中暴露的 Windows 窗口可靠性、沉浸式播放体验、本地/远程曲库存储模型、歌词同步和社区版能力边界整理。

社区分支不包含商业授权层，曲库和歌单能力默认开放。

## 主要修复与改进

### Windows 沉浸式播放

- 修复窗口化进入沉浸式播放时顶部标题栏覆盖不完整的问题。
- 修复最大化后进入沉浸式播放时底部 Windows 任务栏区域露出、出现底部空白或错位的问题。
- 通过 Tauri 原生窗口模式处理全屏/退出全屏，降低仅靠前端尺寸计算带来的不稳定性。
- 平滑沉浸式播放进入和退出动画，减少退出时播放器区域抽动。
- 修复非最大化窗口关闭沉浸式播放后窗口框架逐次扩大的问题。
- 右上角退出/缩放 CTA 按窗口尺寸和像素密度调整触控命中区，改善小尺寸高分屏可点击性。

### Win10 22H2 启动透明兜底

- 加固 Windows 启动阶段的玻璃/透明背景检测与兜底。
- 在系统组合效果异常或 Win10 22H2 出现透明启动反馈时，提前切换到稳定背景，避免首屏像空窗口。

### 曲库索引化与缓存边界

- 本地导入改为以索引为核心，不再把用户音频默认复制进托管暂存盘。
- 远端文件的长期方向调整为索引优先，减少本地缓存对用户磁盘的占用压力。
- 将旧的托管存储根目录重新定义为 legacy cache，为后续回收站/存储管理里清理历史本地缓存做准备。
- 索引状态继续接入现有状态机，用户删除源文件后会让对应曲目及时失效，避免展示不可播放的陈旧条目。

### 歌词与播放同步

- 提升沉浸式歌词播放中的时间轴追踪速度，播放状态下会更积极地拉回当前歌词位置。
- 改进内置歌词解析与展示，支持更稳定的中英双行/翻译行组合。
- 调整歌词播放器计时类型和同步逻辑，为后续更精确的歌词动效和可移动歌词胶囊打基础。

### 胶囊歌词准备项

- 继续整理歌词胶囊交互边界，为后续支持用户拖动位置、展开态锁定和恢复默认位置按钮预留空间。
- 本版本不会把这些能力作为完整用户承诺发布；它们属于后续小版本继续推进的体验项。

### TypeScript 精度基线

- 启用 TypeScript 和 `vue-tsc` 类型检查基线。
- 沉浸式播放器、歌词播放器、外部曲库弹窗和部分通用组件完成 TS 化。
- 发布前构建链路增加类型检查，降低 UI 状态和窗口模式参数继续迭代时的回归概率。

### 授权门限预埋

- 新增 Beta 授权模型和限制能力：
  - Free：最多 3 个曲库。
  - Free：最多 10 个用户歌单。
  - Community：默认完整开放。
- Beta 阶段默认保持自由体验，不主动拦截用户。
- 预留后端许可证校验链路，后续可通过服务器返回的许可证状态解锁限制。

## 真实用户日志反馈

首发后的真实用户日志显示：

- 本地 915 首曲库导入成功，播放链路稳定，无 error / warn / crash。
- 本地文件切歌响应多在几十毫秒到一百多毫秒，native-file 播放链路健康。
- 胶囊歌词反复打开/关闭稳定。
- 远端 WebDAV 首次播放 resolve/cache 偏慢，需要后续继续优化为索引优先和更轻量的首播准备。

## 已知后续项

- ASIO 专用输出不在 `26.0.1` 支持声明内；当前 Windows 输出仍走系统音频设备后端。
- 旧版本产生的本地缓存需要在后续存储管理/回收站里提供更清晰的一键清理入口。
- 诊断日志中的音频设备名称后续应进一步脱敏，默认只上传设备数量、默认设备状态、匿名 hash 和外接设备概况。
- `update_check_skipped` 和胶囊窗口调试事件仍需要继续降噪，避免线上诊断报告被低价值 info 事件刷屏。
